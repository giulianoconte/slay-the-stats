Supports ModConfig (https://www.nexusmods.com/slaythespire2/mods/2).

Color blind setting available.

------------------------------------------------------------------------
WHAT IT DOES
------------------------------------------------------------------------

SlayTheStats reads your full run history retroactively. Stats are updated when you finish a run or start the game. Abandoned runs are skipped.

- Pick% -- how often you pick a card when it's offered as a fight reward.
- Win% -- how often you win runs that include a given card or relic.
- Runs -- shown as a fraction of runs the card was in your final deck / runs it appeared on a fight reward screen (e.g. 12/30). For relics, just the number of runs present. It is split out by act - the act in which you acquired your first copy of the card.
- Buys -- for colorless cards and relics: shown as a fraction of runs you purchased the item from a shop / runs it appeared in a shop (e.g. 12/30).

Stats are shown as a tooltip when you hover over cards and relics during a run, in shops, and in the compendium.

Color coding helps you read the data at a glance:
- Pick% and Win% are colored relative to your personal baseline. Your baseline is the average win rate and pick rate on fight rewards for the character.
- Color intensity is sensitive to both sample size and deviation, so a high winrate over a few runs can be considered as significant as a moderately high winrate across many runs.

------------------------------------------------------------------------
INSTALLATION
------------------------------------------------------------------------

1. Install GUMM for STS2 if you haven't already: https://sts2mods.com/godot-universal-mod-manager-for-sts-2/
2. Download and extract BaseLib (v0.2.5 or later) into your mods folder if you haven't already: https://www.nexusmods.com/slaythespire2/mods/103
3. Download SlayTheStats-vX.X.X.zip from the Files tab and extract it into your mods folder -- you should end up with a SlayTheStats/ folder inside mods/ containing SlayTheStats.dll, SlayTheStats.json, SlayTheStats.pck, and a fonts/ subfolder

------------------------------------------------------------------------
COMPENDIUM FILTERS
------------------------------------------------------------------------

Open the in-game compendium (Card Library or Relic Collection) and click the SlayTheStats button in the sidebar to open the filter pane. The pane lets you tune which runs feed your stats:

- Ascension range -- show stats only from runs within an ascension band e.g. A5+.
- Version range -- restrict to specific game versions.
- Profile -- restrict to a single save profile, or aggregate across all profiles.
- Class -- "All", "Match class card" (use the card's owning class for class cards, all-chars for colorless/curse), or filter to a specific character.
- Group card upgrades -- merge upgraded variants (e.g. Strike+ counts toward Strike) or track them separately.

Active filters are highlighted in green so it's obvious which controls are diverging from your defaults. Three action buttons at the bottom:

- Clear Filters -- remove all filters
- Reset -- restore to your saved defaults
- Save Defaults -- persist the current values as your new defaults. Your defaults are always used when in-run.

------------------------------------------------------------------------
NOTES
------------------------------------------------------------------------

Pick% is sourced only from fight rewards. Shop purchases, event cards, and other sources are not counted.

Win% counts all runs in which the card or relic was present in your final build, from any source.

Shop purchases (Buys): For colorless cards and relics, the tooltip shows how often you purchased an item relative to how often it appeared in a shop, colored relative to your overall shop buy rate. Note that appearances are counted regardless of whether you could afford the item.

Colorless Cards, Event Cards, Ancient Relics, Curses, etc are all tracked.

Mod settings (color blind mode, in-run stats toggle, master off switch, debug mode) can be configured in-game from BaseLib's mod configuration page or, if you have it installed, the optional ModConfig mod (https://www.nexusmods.com/slaythespire2/mods/2). Filtering by ascension, version, profile, character, or upgrade grouping is done via the filter pane in the compendium -- see COMPENDIUM FILTERS above.

------------------------------------------------------------------------
FAQ
------------------------------------------------------------------------

Q: Why does a card show more "present" runs than "offered" runs (e.g. 5/1)?

A: With "Group card upgrades" off, upgraded and base versions are tracked separately. "Present" counts all runs the card ended up in your deck from any source -- including upgrading the base card at a campfire -- while "offered" only counts fight reward screens. Turn on "Group card upgrades" in the filter pane to merge them. Starter cards and relics (those you begin a run with) will always show 0 for "offered" since they never appear on fight reward screens.


Q: Why is my Win% 100% but the stat is grey?

A: Color intensity reflects both sample size and magnitude of deviation from your baseline. With very few runs, even a perfect rate stays muted -- the color intensifies as evidence builds up.


Q: Why don't I see stats for a card I've used?

A: Open the SlayTheStats filter pane in the compendium and check your filters. 


Q: I'm seeing "no data" for everything.

A: The mod cannot find your run history to populate data. In this case you'll need to find where your game save data is (such as run history). Set "DataDirectory" to the "SlayTheSpire2" folder which the data is stored in. For Windows it is %APPDATA%/SlayTheSpire2. Note that you need to put the full path in there. Please file a bug report so I can fix the defaults!

TROUBLESHOOTING

- Windows: %APPDATA%\Godot\app_userdata\Slay the Spire 2\logs\godot.log
- Linux: ~/.local/share/godot/app_userdata/Slay the Spire 2/logs/godot.log
